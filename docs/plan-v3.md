## 4가지 결정사항 (plan-v3 핵심)

### #1 — Supabase Storage 정상 업로드 (Phase 1~2)
**목표**: 인증 사용자의 이미지(PNG/JPG/WebP, 최대 10MB) 업로드 → Storage 저장 → DB URL 기록 → end-to-end 성공  
**Acceptance Criteria**: 401/403/413/415/500 에러 0건  
**Phase**: 1(진단), 2(설정)

### #2 — 역할 구조 2분할 (Phase 3)
**결정**: owner/팀원/클라이언트 3역할 → **admin/user** 2역할  
**변경 내용**:
- `profiles.role` enum → `('admin','user')`
- `public.is_admin()` SECURITY DEFINER 함수
- 기존 owner → admin, 팀원/클라이언트 → user
- 클라이언트 공유는 `share_tokens` 테이블로 분리 (role 불필요)

### #3 — 4가지 시작 케이스 UI (Phase 4)
**결정**: `/projects/new`에서 2×2 grid로 케이스 선택
- **A** 빈 상태: 행사정보 입력 → Claude API로 표준 제작물 추천
- **B** 엑셀 보유: .xlsx 업로드 → 17컬럼 매칭(fuzzy match) → 누락 항목 표시
- **C** 샘플 디자인: 이미지 업로드 → Claude Vision 슬롯 인식 → 텍스트 prefill
- **D** 텍스트만: 행사명/로고/콘텐츠 → 빈 슬라이드 생성

### #4 — 환경장식물 동의어 통합 (Phase 5)
**결정**: `signage_aliases` 테이블 생성 (alias_name, canonical_name, kind, default_size)  
**7개 동의어**:
| 입력 표기 | 표준명 | 근거 |
|-----------|--------|------|
| 스프링 배너 | X배너 | 표기 변형 |
| 드롭배너 | 세로 현수막 | 세로 형태·규격 동일 |
| 난간배너 | 세로 현수막 | 설치 위치만 다름 |
| 천장배너 | 통천 배너 | 천장 매다는 형태 동일 |
| 빵빠레배너 | 가로등 배너 | 폴에 거는 세로형 |
| 피켓(A4) | A4 | 출력물·규격 동일 |
| 피켓(A3) | A3 | 출력물·규격 동일 |

UX: 입력 자동완성 → sub-line "→ X배너로 인식됩니다" + Toast 알림  
Excel/PPT 출력은 표준명으로.
