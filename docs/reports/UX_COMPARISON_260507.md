## 2. 이번 사이클 적용 항목

### A. 로고 클릭 → 메인 대시보드 이동 (5곳)

**참고**: Notion·Vercel·GitHub의 표준 패턴 — "로고는 항상 home으로 가는 길"

적용 위치:
- `EditorToolbar` (편집창)
- `ProjectInfoClient` (프로젝트 정보)
- `DataDashboard` (데이터 관리)
- `archive/page.tsx` (저장된 제작물)
- 모든 곳에서 `<div>` 로고 → `<Link href="/dashboard">`로 교체
- hover 시 `bg-indigo-500` 으로 명도 변화 → 클릭 가능 시각화
- `title="메인 대시보드로 이동"` 툴팁

### B. 프로젝트 제목 클릭 → 메인 대시보드 (2곳)

**참고**: Vercel 프로젝트 페이지 — 제목 클릭 시 프로젝트 목록으로

적용:
- 편집창 브레드크럼 `{project.name}` span → Link
- 정보 페이지 동일

### C. ESC 키로 모달 닫기 + body 스크롤 잠금

**참고**: Slack·Linear·Notion 모든 모달의 표준 동작

적용:
- `NewProjectButton`에 `useEffect` 추가
- 모달 열린 동안 `keydown` 리스너 → ESC = `handleClose()`
- `document.body.style.overflow = 'hidden'` → 뒤 페이지 스크롤 방지
- `isLoading=true`일 땐 ESC 무시 (작업 진행 중 보호)
- cleanup 시 listener 제거 + overflow 복구

### D. 사이트명 ("MICE 디자인 가이드") 클릭 가능

`/data` 페이지의 사이트명도 hover 인디고 + Link 처리.

## 3. 향후 고려 (이번 사이클에서 제외)

복잡도가 높거나 별도 라이브러리가 필요한 항목:

### 1. Cmd+K / Ctrl+K 명령 팔레트
- **참고**: Linear·Vercel 표준
- 라이브러리: `cmdk` 또는 `kbar`
- 우선순위: 중간 — 향후 키보드 사용자 비율 늘면 도입

### 2. 토스트 알림 시스템
- **현재**: `alert()` / `console.log()` 사용 (브라우저 기본 다이얼로그)
- **목표**: `react-hot-toast` 또는 자체 구현
- **이점**: alert는 하던 작업 차단·이탈 위험 / 토스트는 비차단 알림
- 우선순위: 높음 — 5~10분이면 도입 가능

### 3. 키보드 단축키 표시
- 메뉴/버튼에 단축키 (Ctrl+S = 저장 등) 작은 회색 텍스트로 표시
- **참고**: Linear의 모든 메뉴 항목 우측 단축키 표시

### 4. 자동 저장 시각 피드백 강화
- 현재: "저장됨" 텍스트만
- 목표: ✓ 마지막 저장 N초 전 (Notion 패턴)

### 5. Skeleton Screen
- 데이터 로딩 중 회색 윤곽 미리보기
- 현재: spinner만

### 6. Confirmation Dialog 통일
- 현재: `window.confirm()` 사용 (브라우저 기본)
- 목표: 자체 컨펌 모달 (스타일 통일·키보드 친화)

## 4. 적용 효과 측정 (질적)

| 시나리오 | 이전 | 이후 |
|---|---|---|
| 편집창에서 메인으로 이동 | "← 대시보드" 작은 텍스트 찾아 클릭 | 로고 또는 제목 클릭 (큰 영역) |
| 모달 닫기 | 우상단 X 또는 취소 버튼 | ESC 키 또는 클릭 (3지선다) |
| 인터랙티브 요소 발견 | 호버해야 알 수 있음 | 호버 색상 변화 명확 |
| 모달 실수 클릭 | 뒤 화면 스크롤 됨 (포커스 유실) | body 잠금으로 안정 |

Fitts's Law 관점:
- 클릭 가능 영역이 **작은 텍스트 1개 → 큰 로고 + 제목 + 텍스트 3개**로 확대
- 평균 마우스 이동 시간 단축 (특히 작업 후 메인 복귀가 빈번)
