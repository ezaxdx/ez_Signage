# AI 학습 파이프라인 — 제작물 리스트 가이드

> 작성: 2026-05-07 (v4.1 명령본 단위 7 + 갱신-B/D)
> 회의 결정 (260507): "디자인 자체는 걷어내고 학습부터" — 도면 + 발주 리스트 + 결과보고서를 학습시켜 추천 정확도 확보.

---

## 1. INPUT (학습 데이터 소스)

| 데이터 | 포함 정보 | 적용 시점 | 진행 |
|---|---|---|---|
| 발주 리스트 (엑셀) | 제작물 종류·규격·수량·재질·장소 | **1차** (현재 진행) | 🟡 회의/전시 등 파트별 분류 진행 중 |
| 결과보고서 (Word/PDF) | 장소 정보·프로그램 정보 | **1차** (현재 진행) | ⏳ 부분 발췌 (전체 너무 큼) |
| 도면 (PDF/이미지) | 장소별 공간 구성·주출입구·동선 | **2차** (관리자 업로드) | 🟡 컨벤션센터·호텔 우선 |
| 사용자 라이브 데이터 | 신규 프로젝트 자동 누적 | **상시** (자동) | ✅ liveStats.ts 동작 중 |

### 1-A. 결과보고서 학습 정책 (회의 결정)

```
전체 보고서 입력 = 너무 큼 (대용량)
→ 부분 발췌만 학습:
  1. 장소 정보 섹션 (행사장명·홀·공간 구성)
  2. 프로그램표 (시간표, 도면과 함께 입력 시 정교)
  3. 도면 (행사장·동선)
```

### 1-B. 자동 누적 학습 (사용자 핵심 지시)

> "앞으로 제작되는 모든 행사는 기존 정보에 자동 업데이트 되어야만함"

- 별도 트리거 없음
- 새 프로젝트 INSERT → liveStats 5분 캐시 만료 후 자동 재집계
- 행사장·발주처·환경장식물·program_parts 모두 누적 통계에 즉시 반영
- 구현: `lib/data/liveStats.ts` (이미 동작 중)

---

## 2. AI가 추출할 데이터 (1차 목표)

- 장소 × 프로그램 조합별 자주 쓰인 제작물 종류·규격·수량
- 예: 코엑스 전시장 vs 코엑스 컨퍼런스홀에서 많이 쓰인 제작물 분포 차이

### 2-A. 장소 단위 정책 (사용자 결정 260507)

- **기본**: 건물 단위 (코엑스, 킨텍스, 롯데호텔)
- **옵션**: 등록 시 "홀 단위 분리" 체크박스
  - 체크하면 venue_halls 테이블에 홀별 데이터 분리 보관
  - 코엑스 전시장 vs 컨퍼런스홀처럼 차이가 큰 경우만 사용

## 3. OUTPUT (학습 결과 → 사용처)

### 3-A. 정확한 정보로 분류 (회의 직접 명시)
- 가능 종류 (X배너·현수막·포디움 등)
- 규격 (mm)
- 수량 (동선배너 등)

### 3-B. 추천 함수
- `recommendByProbability()` — 시드 + 라이브 합산 확률 기반 (현재 동작)
- `recommendDongseonBannerCount()` — 동선배너 수량 룰베이스 (v4.1 신규)
- `findSimilarPastEvents()` — 행사장·발주처·분류 가중점수 (현재 동작)

## 4. 도면 활용 방안

### 4-A. 1차 가능성 확인 (회의 결정 — 클로드 Vision 가능)
- 클로드/Gemini Vision에 도면 입력 → 주출입구 자동 파악 + 동선배너 위치 제안
- 단독 도면도 어느 정도 작동, 결과보고서 + 프로그램표 함께 입력 시 정교

### 4-B. 우선 적용 카테고리 (피드백 명시)
1. **컨벤션센터** (코엑스, 킨텍스, BEXCO, EXCO 등 — 도면 표준화 잘 됨, 사용 빈도 높음)
2. **호텔** (롯데호텔 외 주요 호텔)
3. **전시장 / 야외 / 기타** (3차)

### 4-C. 도면 관리 위치
- 사용자가 신규 등록 요청 → `venue_requests.floor_plan_url`
- 관리자 직접 추가 → `venues.floor_plan_url`
- 자동 학습 큐 진입 → `learning_jobs.source_url`

## 5. 동선배너 수량 산정 룰 (1차 룰베이스)

### 5-A. 룰 (`lib/recommendation/dongseonBanner.ts`)

```
1. 인원 기반: max(4, attendees / 200)
2. 홀 분리 행사장: 선택된 홀 수 × 1
3. 주출입구 메모 ≥ 2개: 추가 1~N개 (메모 내 콤마 분리 카운트)
4. 대형 행사장 (≥3000㎡): 추가 +2
```

### 5-B. 예시

| 행사 | 인원 | 홀 분리 | 주출입구 | 면적 | 추천 수량 |
|---|---|---|---|---|---|
| 소형 컨퍼런스 | 100 | X | 1곳 | 500㎡ | 4개 (최소) |
| 중형 박람회 | 800 | X | 2곳 | 2000㎡ | 4 + 1 = 5개 |
| 대형 전시 | 3000 | ✓ (3홀) | 4곳 | 5000㎡ | 15 + 3 + 3 + 2 = 23개 |

### 5-C. 향후 (다음 사이클)
- Vision API → 좌표 기반 위치 추천 (`recommendDongseonBannerFromFloorPlan` 시그니처만 마련)
- 도면 + 프로그램표 동시 입력 시 시간대별 동선 변경 반영

## 6. 학습 사이클 (사이클 다이어그램)

```
[관리자가 사전 입력]                [사용자 사용]                [관리자 도면 추가]
       │                              │                              │
       ▼                              ▼                              ▼
  SEED_PERFLIST              projects INSERT                  venues + learning_jobs
  SEED_SIGNAGE_TYPES         design_items INSERT              (도면 → 다음 사이클 Vision)
       │                              │                              │
       └──────────────────────────────┼──────────────────────────────┘
                                      ▼
                          liveStats.fetchLiveStats()
                          (5분 캐시 + 동의어 정규화)
                                      │
                                      ▼
                          recommendByProbability()
                          findSimilarPastEvents()
                          recommendDongseonBannerCount()
                                      │
                                      ▼
                       사용자 새 프로젝트 폼·Case A·관리자 KPI
                                      │
                                      └─→ ↑ (피드백 누적)
```

## 8. 후속 검토 사항 (메모)

- 결과보고서 자동 발췌 (장소·프로그램 섹션 자동 추출 모듈 필요)
