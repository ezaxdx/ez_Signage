## 2. 해결하려는 업무 문제

현재 MICE 행사 진행 시 환경장식물(현수막·배너·통천·천정배너 등) 발주 작업은 행사장별 표준 규격을 매번 매뉴얼·도면에서 직접 확인하고, 과거 유사 행사 발주엑셀을 사람의 기억으로 검색해서 카테고리·수량·위치·규격을 수동으로 결정하는 방식이다. PM/실무자가 행사 1건당 1~2일 소요되며, 천정배너·부속시설(라운지·컨퍼런스장) 사인 등은 신입의 경우 자주 누락한다.

```text
- 매번 행사장 매뉴얼·도면을 다시 확인 (반복 작업)
- 과거 유사 행사 사례 검색이 담당자의 기억에 의존
- 천정배너·부속시설 사인은 누락 빈번 (현장 재출력 비용 발생)
- 신입·이직자가 즉시 발주 못 함 (지식 비대칭)
- 외주 디자이너에게 명확한 발주 가이드 부재 시 시안 재작업 발생
```

## 3. 사용 전 업무 프로세스

### 3.1 현재 업무 흐름

```text
1. 행사 정보 수령 (행사장·일자·참가자 수·격)
2. 해당 행사장 매뉴얼 PDF / 도면 jpg 직접 확인
3. 과거 유사 행사 폴더에서 발주엑셀 사람 기억으로 검색
4. 카테고리별 환경장식물 수량·규격·위치를 수동 추정
5. 17컬럼 발주엑셀 직접 작성
6. PPT 디자인 의뢰 가이드 별도 작성
7. 외주 디자인 업체에 엑셀+PPT 전달
```

### 3.2 현재 사용하는 도구 / 문서 / 시스템

- 행사장별 매뉴얼 PDF (예: `8.전시주최자매뉴얼.pdf`)
- 행사장 도면 jpg (예: `1전시장 5홀.jpg`, `시설개요jpg.jpg`)
- 과거 행사 폴더 (예: `환경장식물 행사별/[핵심] 높음이상/...`)
- 과거 발주엑셀 (예: `20221027 스마트국토엑스포_작물시트만...xlsx`)
- 결과보고서 PDF·HWP
- 사내 G드라이브 공유 폴더 (행사별 자료)

### 3.3 현재 업무의 병목

- 행사장별 표준 규격을 매번 매뉴얼에서 직접 확인 (반복)
- 과거 발주 사례 검색이 사람 기억·G드라이브 폴더 탐색에 의존
- 천정배너 위치·규격을 수동으로 결정 (정답이 매뉴얼에 묻혀있음)
- 부속 시설(라운지·컨퍼런스장·VIP룸) 사인물 누락
- 외부 게이트별 광고 규격 등 세부 정보는 매뉴얼 깊이 들어가야 함

### 3.4 현재 업무의 소요 시간 / 반복 빈도

- **소요 시간**: 행사 1건당 1~2일 (PM 기준)
- **반복 빈도**: 월 5~10회 (AXDX팀 행사 건수 기준)
- **주 사용자**: 행사 PM (대리·사원급)
- **업무 발생 시점**: 행사 3~4주 전 (발주 단계)

### 3.5 현재 방식의 리스크

- **누락 가능성**: 천정배너·부속시설 사인 누락 (월 1~2건 발생)
- **품질 편차**: 담당자 숙련도에 따라 발주 정확도 큰 차이
- **보안 이슈**: 행사장 매뉴얼·도면 외부 노출 위험 (외주 디자인 업체 공유 시)
- **재작업 가능성**: 발주 누락 시 현장 재출력 (1건당 추가 50~200만원)
- **담당자 의존도**: 베테랑 1~2명이 노하우 독점 → 휴가·이직 시 업무 마비

## 4. 사용 후 기대 프로세스

### 4.1 사용 후 업무 흐름

```text
1. 웹앱(/projects/new)에서 행사 정보 입력 (행사장 드롭다운 + 격·인원·국제·VIP 등)
2. AI가 학습된 행사장별 데이터·매뉴얼 표준·과거 사례 기반 발주 리스트 자동 추천
3. 추천 결과를 그리드에서 검토 (학습 데이터 없는 카테고리는 ′추천 없음 + 매뉴얼 보강 권고′ 자동 표시)
4. 누락된 부분은 사용자가 그리드에서 직접 보강 (1회)
5. 발주 전 점검(Preflight) 18가지 자동 검증
6. 17컬럼 발주엑셀 + 14컬럼 PPT 동시 다운로드
7. 외주 디자인 업체에 그대로 전달
```

### 4.2 줄어드는 작업

- 행사장 매뉴얼·도면 매번 직접 확인 (자동 학습 데이터로 대체)
- 과거 행사 발주엑셀 사람 기억 검색 (가중치 자동 매칭으로 대체)
- 카테고리별 수량·규격 수동 추정 (AI 추천으로 대체)
- PPT 디자인 의뢰 가이드 별도 작성 (자동 생성)

### 4.3 새로 생기는 작업

- AI 추천 결과 검수 (1회, 약 30분)
- 학습 데이터 없는 카테고리 매뉴얼 보강 (천정배너 등, 케이스별 발생)
- 신규 행사장 등록 시 매뉴얼·도면 학습 자료 업로드 (/admin/learning)

### 4.4 기대 효과

- **시간 절감**: 행사 1건당 1~2일 → 2~3시간 (약 80% 단축)
- **품질 향상**: 천정배너·부속시설 사인 누락 0건 목표 (학습 데이터 완비 후)
- **누락 방지**: 발주 전 점검 18가지 자동 검증
- **표준화**: 행사장별 매뉴얼 표준 규격 자동 반영 (예: 킨텍스 1전시장 5홀 7600×2000)
- **사용 편의성**: 신입도 즉시 사용 가능 (지식 비대칭 해소)
- **기타 효과**: 베테랑 노하우의 시스템화 / 외주 디자이너 시안 재작업 감소

## 6. 사용자 플로우

```text
1. 웹앱 로그인 (이메일/PW, Supabase Auth)
2. 신규 프로젝트 만들기 → 3단계 위자드
   ├─ 1단계: 프로젝트명·발주처·행사장(드롭다운)·일자
   ├─ 2단계: 프로그램 파트 다중 선택 (전시·공식·컨퍼런스·글로벌·공통운영)
   └─ 3단계: AI 추천 결과 확인 + 수량 조정 + 시안 업로드(선택)
3. 에디터(/projects/[id])에서 그리드+캔버스로 세부 편집
   ├─ 학습 데이터로 자동 채워진 항목은 그대로 통과
   └─ ′추천 없음′ 항목은 매뉴얼 참조해 직접 보강
4. 발주 전 점검(/preflight) 18가지 자동 검증
5. 발주 파일 생성 버튼 → Excel + PPT 동시 다운로드
6. 외주 디자인 업체에 그대로 전달
```

## 7. 입력 데이터

사용자가 입력해야 하는 정보:

- 프로젝트명
- 발주처 (datalist 자동완성)
- 행사장 (드롭다운 + 신규 요청 모달)
- 행사 일자 (시작·종료)
- 참가자 수
- 행사 격 옵션 (국제·VIP·야외·예산제약)
- 행사 언어 (한국어·영어·이중·기타)
- 프로그램 파트 다중 선택 (40.04~40.20 12종)

### 입력 예시

```text
프로젝트명: 2025 스마트국토엑스포
발주처: 한국토지주택공사
행사장: 킨텍스 제1전시장 5홀
일자: 2025-10-15 ~ 2025-10-17
참가자: 5,000명
격: 국제 + VIP
언어: 이중 (한국어·영어)
프로그램 파트: 전시·공식행사·컨퍼런스·글로벌·공통운영
```

### 입력하면 안 되는 정보

- 개인정보 (참가자 명단·연락처)
- 외부 발주처 비밀정보 (계약 금액·내부 결재 사항)
- 미확정 정보 ("[확인 필요]" 표시 없이 단정형으로 입력 금지)

### 입력 데이터 보안 등급 / 주의사항

- **개인정보 포함 여부**: 없음 (행사 메타데이터만)
- **회사 내부자료 포함 여부**: 있음 (발주처·행사 기획 정보)
- **외부 AI 서비스 입력 가능 여부**: Gemini API 호출 시 행사 메타데이터만 전송 (개인정보·계약금액 제외)
- **권한 제한 필요 여부**: Supabase RLS로 프로젝트별 row 수준 접근 제어 + `allowed_users[]`

## 8. 처리 흐름

```text
1. 사용자 입력 정보 받기 (행사 메타데이터)
2. 같은 행사장 과거 사례 검색 (findSimilarPastEvents 가중치 점수)
   - 행사장 +4 / 발주처 +5 / 분류 +3 / PM부서 +2 → 상위 5건
3. 매뉴얼 표준 규격 적용 (venue_specs 시드 데이터)
   - 예: 킨텍스 1전시장 5홀 외벽 7600×2000mm
4. 천정배너·부속시설 패턴 적용 (ceiling_banner_patterns)
5. Gemini 2.5 Flash API 호출 (responseMimeType: application/json)
   - 시스템 프롬프트: MICE 환경장식물 배치 전문가
   - 사용자 프롬프트: 위 1~4 종합 컨텍스트
6. JSON 추천 결과 파싱 (recommendations[], constraints_applied[], confidence)
7. 17컬럼 발주엑셀 + 14컬럼 PPT 변환 (ExportService)
8. 사용자 검수 (그리드 편집 가능)
9. 발주 전 점검(Preflight) 18가지 자동 검증
10. 최종 다운로드
```

## 9. AI 적용 지점

| 단계 | AI 역할 | 설명 |
|---|---|---|
| 1 | 분류 | 행사 격(국제·VIP·참가자 수)·프로그램 파트로 권장 환경장식물 카테고리 자동 분류 |
| 2 | 추천 | 학습된 행사장별 매뉴얼 표준 + 과거 사례 + 도면 분석으로 카테고리·수량·위치·규격 추천 |
| 3 | 검토 | 학습 데이터 없는 카테고리는 ′추천 없음 + 매뉴얼 보강 권고′ 자동 표기 (환각 방지) |
| 4 | 리스크 탐지 | 발주 전 점검 18가지 자동 검증 (누락·중복·규격 오류 등) |

AI 역할 매핑:

```text
분류 → 행사 격 분류, 프로그램 파트별 권장 환경장식물
요약 → 과거 유사 행사 5건을 추천 컨텍스트로 압축
초안 작성 → 17컬럼 발주엑셀 행 자동 생성
추천 → 카테고리·수량·위치·규격
검토 → 학습 데이터에 없는 카테고리는 추천하지 않음
변환 → JSON → Excel + PPT
검색 → findSimilarPastEvents (가중치 점수)
질의응답 → (현재 미지원, 향후 챗봇 검토)
리스크 탐지 → Preflight 18가지
```

## 10. 출력물

최종적으로 사용자가 받는 결과물:

- **발주엑셀** (17컬럼): NO / 파트 / 구분 / 장소 / 사용목적 / 품목 / 언어 / 규격 / 재질 / 수량 / 내용 / 비고 / 담당자 / 디자인업체 / 출력업체 / 설치시간 / 철거시간
- **디자인 의뢰 가이드 PPT** (14컬럼, 1 환경장식물 = 1 슬라이드, 13.3×7.5 inch)
- **자동 누락 표기**: 학습 안 된 카테고리는 ′[추천 없음 - 매뉴얼 보강 필요]′ 비고에 자동 표시
- 파일명: `제작물리스트_{행사명}_{YYYYMMDD}.xlsx` / `{행사명}_디자인의뢰가이드_{YYYYMMDD}.pptx`

### 출력 예시

```text
NO.   파트     구분        장소         사용목적      품목       언어  규격(mm)        재질   수량  내용
```
