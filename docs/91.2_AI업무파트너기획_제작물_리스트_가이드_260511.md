## 2. 해결하려는 업무 문제

```text
사원이 겪는 문제:
각 행사의 환경장식물을 제작하는 경우 각 행사별로 환경장식물이 몇 개가 필요한지,
어느 위치에 환경장식물이 들어가는지 판단하기 어려운 문제가 있음. 이를 해결하고자 함.

이 문제가 해결되지 않을 시 환경장식물별 맥락 파악의 어려움으로 인해
행사별 환경장식물 리스트를 작성할 시 정확하게 작성하지 못하는 리스크가 있음.
기존 행사에 환경장식물을 얼마나 사용했는지 자료를 수집하는 비효율이 생김.

같은 정보로 엑셀, PPT 2가지 자료를 만들어야 하므로 편집 진행의 비효율이 발생.
```

## 3. 사용 전 업무 프로세스

1. 각자 파트에서 필요한 환경장식물 리스트를 보내준다.
2. 리스트를 한 개의 엑셀에 모은다.
3. 모은 엑셀을 PPT에 한 장당 한 행씩 넣는다.
4. 각 PPT에 환경장식물을 입력하여 예시를 작성한다.
5. 디자인 협력사에게 환경장식물 시안 요청한다.
   - 요청할 때 키비주얼 등 소스 보내준다.
6. 시안에서 피드백을 기입하여 협력사에 수정 요청을 한다.
7. 시안 수정 반복 후 확정한다.
8. 협력사에게 최종 엑셀 혹은 PPT 리스트 보내준다.
9. 리스트대로 시안 뽑아준다.
10. 환경장식물 출력 협력사에게 시안 보내준다.
11. 잘 나왔는지 검수 필수.
12. 출력 협력사와 현장에 몇 시까지 설치/철거할지 정해놓는다.
13. 설치날 잘 설치하고 있는지 관리한다.

## 4. 사용 후 기대 프로세스

환경장식물 취합 시 제공한 가이드를 확인하여
환경장식물 배치의 감각이 없어도 쉽게 감각이 있는 사람처럼 배치 가능.

엑셀, PPT를 한 번에 제작하여 편집의 비효율이 감소.

6. 시안 수정 반복 후 확정한다.
7. 협력사에게 최종 엑셀 혹은 PPT 리스트 보내준다.
8. 리스트대로 시안 뽑아준다.

### 입력 예시

```text
프로젝트명: 2025 스마트국토엑스포

참가자: 5,000명
프로그램 파트: 전시·공식행사·컨퍼런스·글로벌·공통운영
```

### 입력하면 안 되는 정보

## 8. 처리 흐름

```text
1. 사용자 입력 정보 받기 (행사 메타데이터)
2. 같은 행사장 과거 사례 검색
   - 행사장 / 분류 기반 검색
3. 행사장 매뉴얼 표준 규격 적용
   - 예: 킨텍스 1전시장 5홀 외벽 7600×2000mm
4. Gemini 2.5 Flash API 호출 (responseMimeType: application/json)
   - 시스템 프롬프트: MICE 환경장식물 배치 전문가
   - 사용자 프롬프트: 위 1~3 종합 컨텍스트
5. JSON 추천 결과 파싱
6. 사용자 검수 (그리드 편집 가능)
7. 17컬럼 발주엑셀 + 14컬럼 PPT 변환
8. 최종 다운로드
```

## 9. AI 적용 지점

| 단계 | AI 역할 | 설명 |
|---|---|---|
| 1 | 분류 | 행사 격(국제·VIP·참가자 수)·프로그램 파트로 권장 환경장식물 카테고리 자동 분류 |
| 2 | 추천 | 학습된 행사장별 매뉴얼 표준 + 과거 사례 + 도면 분석으로 카테고리·수량·위치·규격 추천 |
| 3 | 검토 | 학습 데이터 없는 카테고리는 '추천 없음 + 매뉴얼 보강 권고' 자동 표기 |
| 4 | 리스크 탐지 | 엑셀·PPT 다운로드 전 자동 검증 (행사장 매뉴얼 표준 규격 등 추천 기반 탐지) |

## 10. 출력물

최종적으로 사용자가 받는 결과물:

- **발주엑셀** (17컬럼): NO / 파트 / 구분 / 장소 / 사용목적 / 품목 / 언어 / 규격 / 재질 / 수량 / 내용 / 비고 / 담당자 / 디자인업체 / 출력업체 / 설치시간 / 철거시간
  - 파일명: `제작물리스트_{행사명}_{YYYYMMDD}.xlsx`
- **디자인 의뢰 가이드 PPT** (14컬럼, 1 환경장식물 = 1 슬라이드, 13.3×7.5 inch)
  - 파일명: `{행사명}_디자인의뢰가이드_{YYYYMMDD}.pptx`
- **자동 누락 표기**: 학습 안 된 카테고리는 `[추천 없음 - 매뉴얼 보강 필요]` 비고에 자동 표시

### 출력 예시

```text
NO.   파트     구분        장소         사용목적      품목       언어  규격(mm)        재질   수량  내용
01    공통운영 가로현수막  킨텍스 1F   외부 메인     기둥현수막  EN/KO  12000x1000     현수막 1     행사타이틀
```

### 출력물 사용 방식

- 검토 후 사용하는가: 예 (그리드에서 1회 검수 + 발주 전 점검)
- 다른 문서에 붙여넣는가: 아니오 (Excel·PPT 그대로 사용)
- 별도 시스템에 업로드하는가: 외주 디자인 업체 이메일·드라이브에 전달

### 출력물 품질 기준

- 정확성: 정답지 대비 수량 ±20% 이내 / 위치 매칭률 60% 이상 / 규격 9종 중 7종 이상 일치
- 형식 준수: 17컬럼 표준 / mm 단위 / 한국어 + 영문 병기

### 화면 UI

- 화면 백그라운드 컬러 변경: 화이트 또는 밝은 컬러

### 추가 피드백

1. **센터별(장소별) 환경장식물 리스트와 시설 가이드를 함께 반영 (사전 학습)**
   - 해당 센터에서 무엇을 설치할 수 있고 무엇을 못하는지 가이드를 줄 수 있도록
     - 리깅 가능 여부, 하중 제한, 설치 조건, 디지털 사이니지 허용 범위처럼 현장 규정과 안전 기준 포함
   - ★ 제작물 리스트 가이드의 최종 목적이 결과물 생성 자체가 아니라 실제 발주와 설치가 가능한 형태로 안내하는 것
   - 포함될 수 있는 정보:
     - 센터별 설치 가능 항목
     - 리깅 가능 여부와 하중 제한
     - 시설별 주의사항과 금지조건
     - 디지털 사이니지와 같은 확장 항목 규정
     - 센터 사이트나 시설 가이드에 있는 참고 URL 또는 안내 정보

2. **입력 데이터 축적**
   - 출력 내용 (완료 시)에만 재학습하는 것이 아니라 입력 결과도 축적해야 함

3. **발주 가능한 형태의 가이드로 디벨롭** → 설치일자, 철거일자 등
   - 현행 문제점: 사람이 계속 변경되어 발주 행위 자체의 품질이 낮음
   - 발주 의뢰서 형식에 가까운 산출물이 나와야 함
   - 사원급 사용자 고려 → 제작물을 언제까지 컨펌 받아야 하고 언제까지 발주해야 하는지 가이드를 줄 필요성 필요
     - 작성 컬럼이 늘어나서 현재는 삭제한 상황으로 별도 레이어나 안내창으로 안내하는 등의 방법 고려
