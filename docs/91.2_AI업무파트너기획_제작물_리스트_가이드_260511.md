# 제작물 리스트 가이드

## 1. 기본 정보

- AI 업무 파트너명: 제작물 리스트 가이드
- 관련 업무영역: MICE 행사 환경장식물 발주
- 관련 KPI/목표: 발주 오류 방지 / 신입 즉시 투입 가능(기존 환경장식물 제공 내역 제공) / 발주·설치 가능한 형태로 안내
- 개발 유형: 사내구축형
- 주요 사용자: 실무자 (대리·사원급)
- 담당자: 조기흠, 김연아
- 검토자: 김연아
- 운영/유지보수 담당자: 조기흠
- 현재 상태: 프로토타입 (2차 피드백 반영, 2026.05.12 팀 내부 테스트 예정)
- 보고 필요 여부: 예 (5/12 팀장 컨펌 → 사업부 시범 운영 가능 판단 시 이사님 컨펌 → 사업부 공식 요청 메일)
- 팀 공유 필요 여부: 예
- 관련 태그: `#환경장식물` `#MICE` `#발주자동화` `#Gemini` `#학습기반추천` `#시설가이드`

## 2. 해결하려는 업무 문제

```text
사원이 겪는 문제:
각 행사의 환경장식물을 제작하는 경우 각 행사별로 환경장식물이 몇 개가 필요한지,
어느 위치에 환경장식물이 들어가는지 판단하기 어려운 문제가 있음. 이를 해결하고자 함.

이 문제가 해결되지 않을 시 환경장식물별 맥락 파악의 어려움으로 인해
행사별 환경장식물 리스트를 작성할 시 정확하게 작성하지 못하는 리스크가 있음.
기존 행사에 환경장식물을 얼마나 사용했는지 자료를 수집하는 비효율이 생김.

같은 정보로 엑셀, PPT 2가지 자료를 만들어야 하므로 편집 진행의 비효율이 발생.

★ 현행 문제점 (2차 피드백):
사람이 계속 변경되어 발주 행위 자체의 품질이 낮음.
실제 발주·설치가 가능한 형태로 안내가 필요 (단순 리스트 생성이 아닌
발주 의뢰서 형식의 산출물 + 컨펌·발주 마감 일정 가이드).
```

## 3. 사용 전 업무 프로세스

### 3.1 현재 업무 흐름

```text
1. 각자 파트에서 필요한 환경장식물 리스트를 보내준다
2. 리스트를 한 개의 엑셀에 모은다
3. 모은 엑셀을 PPT에 한 장당 한 행씩 넣는다
4. 각 PPT에 환경장식물을 입력하여 예시를 작성한다
5. 디자인 협력사에게 환경장식물 시안 요청한다 (요청 시 키비주얼 등 소스 첨부)
6. 시안에서 피드백을 기입하여 협력사에 수정 요청
7. 시안 수정 반복 후 확정
8. 협력사에게 최종 엑셀 혹은 PPT 리스트 전달
9. 리스트대로 시안 뽑아준다
10. 환경장식물 출력 협력사에게 시안 전달
11. 잘 나왔는지 검수 필수
12. 출력 협력사와 현장에 몇 시까지 설치/철거할지 확정
13. 설치날 잘 설치하고 있는지 관리
```

### 3.2 현재 사용하는 도구 / 문서 / 시스템

- 엑셀, PPT
- 사내 G드라이브 공유 폴더 (행사별 자료)
- 행사장별 매뉴얼·도면 (각 행사장 공식 자료)
- 과거 행사 발주엑셀 (담당자 PC·드라이브 분산)

### 3.3 현재 업무의 병목

- 환경장식물 배치는 감각적인 부분 — 사원들은 알기 어려움
- 같은 정보로 엑셀·PPT 2가지 자료를 만들어야 하므로 편집 진행의 비효율
- 행사장·매뉴얼·도면을 매번 직접 확인하는 반복 작업
- 과거 발주 사례 검색이 담당자 기억·G드라이브 폴더 탐색에 의존
- 천정배너·부속시설(라운지·컨퍼런스장) 사인물 누락 빈번
- 행사장별 설치 가능 여부(리깅·하중·디지털 사이니지)를 매번 확인

### 3.4 현재 업무의 소요 시간 / 반복 빈도

- **소요 시간**: 행사 1건당 1~2일 (PM 기준, 환경장식물 취합만)
- **반복 빈도**: 월 5~10회 (AXDX팀 행사 건수 기준)
- **주 사용자**: 환경장식물 취합 담당자 (대리·사원급)
- **업무 발생 시점**: 행사 3~4주 전 (발주 단계) + 설치·철거 시점

### 3.5 현재 방식의 리스크

- **누락 가능성**: 천정배너·부속시설 사인·디지털 사이니지 항목 누락 (월 1~2건)
- **품질 편차**: 담당자 숙련도 차이로 발주 정확도 큰 격차
- **보안 이슈**: 행사장 매뉴얼·도면 외부 노출 위험 (협력사 공유 시)
- **재작업 가능성**: 발주 누락 시 현장 재출력 (1건당 50~200만원 추가)
- **담당자 의존도**: 베테랑 1~2명이 노하우 독점, 휴가·이직 시 업무 마비
- **설치 가능 여부 미확인**: 리깅 불가 / 하중 초과 시 현장 제거 위험

## 4. 사용 후 기대 프로세스

환경장식물 취합 시 제공한 가이드를 확인하여 환경장식물 배치의 감각이 없어도 쉽게 감각이 있는 사람처럼 배치 가능. 엑셀, PPT를 한 번에 제작하여 편집의 비효율이 감소. 실제 발주·설치가 가능한 형태의 산출물이 나옴.

### 4.1 사용 후 업무 흐름

```text
1. 제작물 리스트 가이드에서 행사를 생성한다
2. 각자 파트에서 제작물 리스트 가이드의 제공자료(센터별 환경장식물 리스트 + 시설 가이드)를 참고하여 필요한 환경장식물 리스트를 기입한다
3. AI 추천 결과를 확인하고 그리드에서 검수
   - 학습 데이터에 있는 항목은 자동 입력
   - 학습 데이터가 없는 항목은 ′추천 없음 + 매뉴얼 보강 권고′ 자동 표기
   - 행사장 설치 가능 여부·리깅·하중 자동 검증
4. 발주 마감일·설치/철거 일자 자동 안내 (사원급 안내 레이어)
5. 디자인 협력사에게 환경장식물 시안 요청 (키비주얼 등 소스 첨부)
6. 시안 피드백 → 협력사 수정 요청
7. 시안 수정 반복 후 확정
8. 협력사에게 최종 엑셀/PPT 전달
9. 출력 협력사에게 시안 전달
10. 검수
11. 설치/철거 시간 확정 (산출물에 자동 포함)
12. 설치날 현장 관리
```

### 4.2 줄어드는 작업

- 제작물 배치 시 기존에 얼마나 배치했는지 레퍼런스 찾는 과정
- 행사장·매뉴얼·도면을 매번 직접 확인
- 엑셀과 PPT 2번 만드는 편집 작업
- 설치 가능 여부 매번 매뉴얼 확인 (리깅·하중·디지털 사이니지)
- 발주·컨펌 마감일 사람이 매번 계산

### 4.3 새로 생기는 작업

- 신규 행사장 등록 시 학습 자료 업로드 (`/admin/learning`)
- 제작물 추천을 위한 행사 정보 입력
- AI 추천 결과 검수 1회 (그리드)

### 4.4 기대 효과

- **시간 절감**: 편집·레퍼런스 조사 감소로 행사당 1~2일 → 2~3시간 (약 80% 단축)
- **표준화**: 기존 행사 정보 제공으로 결과의 표준화 / 행사장별 표준 규격 자동 반영
- **누락 방지**: 천정배너·부속시설·디지털 사이니지 등 자동 인지
- **설치 안전성**: 리깅·하중·디지털 사이니지 허용 범위 자동 검증 (현장 제거 위험 0)
- **사용 편의성**: 신입도 즉시 사용 가능, 컨펌·발주 마감일 자동 안내
- **사람 변경에 강함**: 베테랑 노하우의 시스템화로 담당자 의존도 감소

## 5. 사용자와 사용 상황

- **1차 사용자**: 환경장식물 취합 담당자 (대리·사원급)
- **보조 사용자**: 각 파트 담당자 (파트별 환경장식물 리스트 기입)
- **관리자**: 데이터 학습 관리자 (행사장·매뉴얼·시설 가이드 등록)
- **검토자**: 김연아 / 팀장
- **사용 빈도**: 행사 진행별 1회 (월 5~10회)
- **사용 시점**: 행사의 환경장식물 발주 필요 시 (행사 3~4주 전)
- **사용 장소 / 시스템**: 사내 / 웹앱(데이터허브-AI 게이트)

## 6. 사용자 플로우

```text
1. 웹앱 로그인
2. 신규 프로젝트 만들기 (3단계 위자드)
   ├─ 1단계: 프로젝트명·발주처·행사장(드롭다운)·일자, 엑셀 입력(선택)
   ├─ 2단계: 프로그램 파트 다중 선택
   └─ 3단계: AI 추천 결과 확인 + 수량·이름·규격 변경 + 시안 업로드(선택)
3. 세부 편집 (에디터)
   ├─ 학습 데이터로 채울 수 있는 부분 자동 입력
   ├─ 행사장 시설 가이드 자동 검증 (리깅·하중·디지털 사이니지)
   └─ 그 외 부분 직접 편집하여 보강
4. 컨펌·발주 마감일 안내 확인 (별도 레이어)
5. Excel + PPT 다운로드 (동시 생성)
6. 외주 디자인 업체에 그대로 전달
```

## 7. 입력 데이터

사용자가 입력해야 하는 정보:

- 프로젝트명
- 행사장 (드롭다운 + 신규 요청 모달)
- 행사 일자 (시작)
- 참가자 수
- 프로그램 파트 다중 선택

### 입력 예시

```text
프로젝트명: 2025 스마트국토엑스포
행사장: 킨텍스 제1전시장 5홀
일자: 2025-10-15
참가자: 5,000명
프로그램 파트: 전시·공식행사·컨퍼런스·글로벌·공통운영
```

### 입력하면 안 되는 정보

- 정확하지 않은 정보 (확정되지 않은 행사장·일자 등 단정형 입력 금지)

### 입력 데이터 보안 등급 / 주의사항

- **개인정보 포함 여부**: 없음
- **회사 내부자료 포함 여부**: 행사 기획 정보
- **외부 AI 서비스 입력 가능 여부**: Gemini API 호출 시 행사 메타데이터만 전송 (개인정보·계약금액 제외)
- **권한 제한 필요 여부**:
  - 프로젝트별 담당자만 접근 가능하도록 접근 제어 (`allowed_users[]`)
  - admin에 관한 접근 제어 (`profiles.role = 'admin'`)

### 입력 데이터 축적 정책 (2차 피드백 반영)

★ 출력 완료본 외에 **입력·중간 수정 결과도 학습 데이터로 축적**. 단 미완성·오류 데이터 혼입으로 추천 품질이 떨어지는 위험을 막기 위해 **단계별 분리 저장 + 신뢰도 가중치**를 적용. 상세는 §11-2 참조.

## 8. 처리 흐름

```text
1. 사용자 입력 정보 받기 (행사 메타데이터)
2. 같은 행사장 과거 사례 검색
   - 행사장 / 분류 기반 검색 (findSimilarPastEvents 가중치 점수)
3. 행사장 매뉴얼 + 시설 가이드 적용
   - 예: 킨텍스 1전시장 5홀 외벽 7600×2000mm
   - 리깅 가능 여부 · 하중 제한 · 디지털 사이니지 허용 범위
4. Gemini 2.5 Flash API 호출 (responseMimeType: application/json)
   - 시스템 프롬프트: MICE 환경장식물 배치 전문가
   - 사용자 프롬프트: 위 1~3 종합 컨텍스트 + 시설 가이드
5. JSON 추천 결과 파싱
6. 시설 가이드 자동 검증 (리깅 불가 위치·하중 초과 항목 자동 표시)
7. 사용자 검수 (그리드 편집 가능)
8. 컨펌·발주·설치·철거 마감일 자동 계산 (행사 일자 기준)
9. 17컬럼 발주엑셀 + 14컬럼 PPT 동시 변환
10. 최종 다운로드
```

## 9. AI 적용 지점

| 단계 | AI 역할 | 설명 |
|---|---|---|
| 1 | 분류 | 행사 격(국제·VIP·참가자 수)·프로그램 파트로 권장 환경장식물 카테고리 자동 분류 |
| 2 | 추천 | 학습된 행사장별 매뉴얼 표준 + 과거 사례 + 도면 분석으로 카테고리·수량·위치·규격 추천 |
| 3 | 검토 | 학습 데이터 없는 카테고리는 ′추천 없음 + 매뉴얼 보강 권고′ 자동 표기 |
| 4 | 시설 검증 | 시설 가이드(리깅·하중·디지털 사이니지 규정) 자동 검증, 위반 시 안내 알랏(강제 차단 X, ′그래도 진행′ 선택 가능 — §11-6) |
| 5 | 일정 안내 | 행사 일자 기준 컨펌·발주·설치·철거 마감일 자동 계산 |
| 6 | 리스크 탐지 | Excel·PPT 다운로드 전 자동 검증 (행사장 매뉴얼 표준 규격 기반) |

AI 역할 예시:

```text
분류 → 행사 격 분류, 프로그램 파트별 권장 환경장식물
요약 → 과거 유사 행사 5건을 추천 컨텍스트로 압축
초안 작성 → 17컬럼 발주엑셀 행 자동 생성
추천 → 카테고리·수량·위치·규격
검토 → 학습 데이터에 없는 카테고리는 추천하지 않음
변환 → JSON → Excel + PPT
검색 → findSimilarPastEvents (가중치 점수)
시설 검증 → 리깅 가능 여부·하중 제한·디지털 사이니지 허용 범위
일정 안내 → 컨펌·발주·설치·철거 마감일 자동 계산
리스크 탐지 → 행사장 매뉴얼 표준 규격 등 추천 기반 탐지
```

### 11-2. 입력 데이터 축적 (단계별 분리 + 신뢰도 가중치)

★ **우려 (현장 의견)**: 입력·중간 수정 값은 미완성이라 그대로 학습하면 미완성·오류 데이터가 학습 풀에 섞여 추천 품질이 떨어질 위험이 있음.

★ **해결 원칙**: 데이터의 ′확정 단계′에 따라 **분리 저장**하고 **신뢰도 가중치**로 차등 학습. 미확정 데이터는 개별 추천 모델 학습에는 미사용, 집계 통계만 활용.

#### 11-2-1. 단계별 분리 저장

| 단계 | 저장 형태 (DB) | 신뢰도 가중치 | 학습 활용 범위 |
|---|---|---|---|
| ① 입력 단계 | `inputs` (raw, draft 플래그) | 10% | 행사 메타·발주처 빈도 등 집계 통계만 |
| ② 중간 수정 | `edits` 로그 (수정 패턴 추출용) | 30% | ′이 카테고리는 자주 추가/삭제′ 패턴 |
| ③ 사용자 컨펌 항목 | `confirmed=true` 플래그 | 70% | 항목별 추천 가중치 반영 |
| ④ 발주·다운로드 완료 | `finalized` 테이블 | 100% (정답 풀) | 추천 모델 재학습 핵심 시드 |
| (보너스) 시설 가이드 검증 통과 | `validated_at` 타임스탬프 | +20% | 검증 통과 항목 가중치 추가 |

#### 11-2-2. 운영 규칙

- ′추천 없음′ 항목을 사용자가 **컨펌 플래그 ON**으로 수동 입력한 경우만 학습 시드로 활용
- 미확정 입력(① ②)은 **집계 통계**(빈도·시기 분포·수정 빈도)에만 활용, **개별 추천 모델 학습은 ③ 이상부터**

#### 11-2-3. 우려·반영의 균형 (보고용 정리)

- 사용자가 추가 정보를 가지고 있을 수 있음 (직접 행사장에 문의해 예외 허가 받은 경우 등)

#### 11-6-1. 자동 안내 알랏 (위반 시)

##### 동작 예시

```
[사용자가 ′킨텍스 5홀 천정배너 8000x6000′ 입력]
   ↓
[시스템 검증]
   - 매뉴얼 학습 데이터: 5홀 천정배너 표준 5500×4000 또는 3500×3000
   - 입력값 8000×6000은 표준 외 규격
   ↓
[안내 알랏 표시]
   ┌─────────────────────────────────────────────┐
   │ ⚠️ 시설 가이드 안내                            │
   │                                                │
   │ 킨텍스 1전시장 5홀의 매뉴얼 기준 천정배너 표준    │
   │ 규격은 5500×4000 또는 3500×3000입니다.        │
   │                                                │
   │ 입력하신 8000×6000은 표준 외 규격입니다.       │
   │                                                │
   │ ※ 본 정보는 학습된 시점 기준이며 행사장 정책이  │
   │   변경됐을 가능성이 있습니다.                  │
   │   확신이 없으시면 행사장에 직접 확인 권장.     │
   │                                                │
   │   [수정하기]    [그래도 진행]                  │
   └─────────────────────────────────────────────┘
```

#### 동작 규칙 — ★ 매번 띄우지 않음 (사용자 방해 최소화)

| 상황 | 시스템 동작 |
|---|---|
| 한 행사에서 **첫 시설 가이드 위반** | 안내 알랏 1회 표시 (강제 차단 X) |
| 사용자가 ′수정하기′ 선택 | 입력 필드로 돌아가 수정 가능 |
| 사용자가 ′그래도 진행′ 선택 | 입력값 저장 + 비고에 ′[시설 가이드 외] 사용자 확인 진행′ 자동 표기 |
| **2번째 이후 위반 입력** | 알랏 팝업 ❌ / 그리드 셀에 **⚠️ 아이콘 상시 표시** (호버 시 상세 풍선) |
| 동일 항목 재입력 시 | 알랏 표시 ❌ (사용자 의식적 결정으로 간주) |
| **최종 다운로드 직전** | "시설 가이드 외 항목 N건 — 그래도 진행하시겠습니까?" 일괄 요약 1회 + 그대로 다운로드 가능 |

#### ★ 알림 강도 3단 토글 (프로젝트별 설정)

| 모드 | 알랏 팝업 | ⚠️ 아이콘 | 다운로드 직전 요약 | 적용 상황 |
|---|---|---|---|---|
| `verbose` (기본값) | ✅ (첫 위반 1회) | ✅ 상시 | ✅ | 처음 사용 / 신입 사용자 |
| `silent_icon` | ❌ | ✅ 상시 | ✅ | 베테랑·반복 작업자 |
| `off` | ❌ | ❌ | ✅ (이것만 유지) | 시스템 정보 불신 / 사용자가 모두 알고 작업 |

- 사용자가 에디터 우상단 `시설 가이드 알림: [verbose ▼]` 셀렉터에서 변경
- 설정은 **프로젝트별로 저장** (`project_column_settings`에 함께)
- `off`로 둬도 **다운로드 직전 일괄 요약은 항상 1회** — 최소 안전망 (그것도 한 번 더 끌지는 별도 결정)

#### 핵심: ′매번′이 아니라 ′의식적 결정 1회 + 시각 표시 상시 + 사용자 강도 선택′

- 알랏 팝업은 **첫 위반 시 1회만** (verbose 모드 기준)
- 그리드 셀 **⚠️ 아이콘**으로 상시 시각 표시 — 작업 흐름 방해 없음
- 사용자가 알림 강도를 **3단(verbose / silent_icon / off)**으로 직접 조절
- 어떤 모드든 **다운로드 직전 일괄 요약**은 유지 (안전망)

#### ′그래도 진행′ 항목의 학습 처리

- 사용자가 ′그래도 진행′으로 통과시킨 항목은 **′예외 케이스′ 태그**로 별도 저장
- 같은 행사장·카테고리에서 예외 케이스가 누적되면 → 자동 알림: "이 행사장 ○○ 항목은 표준 외 발주가 잦음. 매뉴얼 업데이트 필요?"
- 관리자(`/admin/learning`)가 검토 후 매뉴얼 데이터 갱신 결정

#### 11-6-2. 시설 가이드 수동 조회 (필요할 때 사람이 호출)

## 14-A. 시스템 핵심 동작 목표 — 점진적 정확도 향상 (★ v9 신규)

> **윗선 명문화 (2026-05-11)**: "우리의 정보는 기존 데이터 + 신규 데이터(앱 사용 데이터)를 통해 처음엔 기본적인 정확도를 제공하지만 향후 점점 더 정확한 정보를 제공하는 것이 목표. 즉, AI를 향후 사용한다면 사용자가 특정 장소와 특정 행사를 골랐을 시 해당 쌓여있는 정보들을 AI에게 제공하여 분석하여 최종 아웃풋을 제공하도록 하는 것이 목표 사항."

### 14A-1. 3단계 정확도 곡선

| 단계 | 시점 | 데이터 소스 | 기대 정확도 |
|---|---|---|---|
| **A. 초기** | 사용 시작 직후 | 시드(과거 발주엑셀 14건 + 폴더 137개 + 동의어·재질 시드) | 60~70% (행사장 학습 보유 시 80%+) |
| **B. 운영 누적** | 사용 1~3개월 | 시드 + 입력(10%)·중간(30%)·컨펌(70%)·발주완료(100%) 가중 누적 | 80~90% |
| **C. 성숙** | 사용 6개월+ | 같은 행사장 발주완료 항목이 다수 누적된 상태 | 90%+ (정답 풀이 충분) |

### 14A-2. ′장소 + 행사 유형 선택 → AI 호출′ 흐름

```
[사용자] 신규 프로젝트 생성
   └─ 행사장 드롭다운에서 선택 (예: 킨텍스 제1전시장 5홀)
   └─ 행사 유형 다중 선택 (예: 전시 + 공식행사)
        ↓
[시스템] 자동으로 누적 컨텍스트 수집
   ├─ Seed: dashboardSeed.ts (과거 17건 실적 + 281건 실측)
   ├─ Seed: venueSignageHelper.ts (발주엑셀 14건 통합 분석)
   └─ App 누적: accumulatedContext.ts ← v9 신설
        └─ Supabase에서 같은 행사장 매칭 프로젝트 5건의 design_items 전부 끌어옴
        └─ 단계별 가중치 부여 (입력 10%·중간 30%·컨펌 70%·완료 100%)
        └─ 카테고리 분포 + 상위 8개 항목 압축
        ↓
[Gemini 2.5 Flash] 모든 컨텍스트를 단일 프롬프트로 받아 분석
   ├─ Seed (행사 유형별 권장 구성)
   ├─ 과거 유사 행사 5건 (행사장·발주처·분류·PM부서 가중)
   ├─ 같은 행사장 발주엑셀 1~3건
   └─ 앱 누적 데이터 (가중치 70%+ 항목은 ′정답에 가까움′으로 표기)
        ↓
[출력] 환경장식물 추천 리스트
   ├─ 가중치 70%+: 그대로 채택 (높은 신뢰도)
   ├─ 가중치 10~30%: 패턴 참고 (검토 권고)
   └─ 학습 데이터 부재 카테고리: ′추천 없음 + 매뉴얼 보강′ 표기 (환각 금지)
```

### 14A-3. 자가 강화 루프

```
사용자 발주 완료 (100% 가중치 데이터 신설)
  ↓
다음 사용자 같은 행사장 추천 호출
  ↓
이전 발주 완료본이 ′정답에 가까움′ 컨텍스트로 자동 주입
  ↓
추천 정확도 점진적 상승
```

→ **별도 학습 트리거 없음**. `confirmed=true` 또는 `finalized_at` 컬럼만 채우면 자동 누적 (`accumulatedContext.ts`의 가중치 분류기가 즉시 인식).
