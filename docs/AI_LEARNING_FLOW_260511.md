# AI 적용 흐름 — 회의록 2026-05-11 기반 정리

> 회의 결정: AI는 사용자에게 보이지 않게 백엔드에서 동작. 사용자 UI엔 ′AI/추천′ 같은 문구 노출 최소화.

## 1. AI 투입 지점 (2가지)

| # | 지점 | 역할 |
|---|---|---|
| ① | **학습** | 행사장·완료본·예외 입력을 텍스트 형태로 누적 |
| ② | **계산** | 사용자가 행사장+행사 유형 선택 시 누적 텍스트를 컨텍스트로 받아 ′어떤 환경장식물을 몇 개 제공할지′ 결정 |

′지점 ②′가 최종 출력 (제작물 리스트) 산출.
′지점 ①′은 ②의 입력 품질을 점진적으로 끌어올리는 백그라운드 작업.

## 2. 학습 3종 (회의 결정)

| 단계 | 트리거 | 입력 데이터 | 산출 |
|---|---|---|---|
| **A. 신규 행사장 등록** | `/admin/learning` 또는 사용자 ′신규 등록 요청′ 후 관리자 승인 시 | 행사장 도면(JPG/PDF) + 권역·유형·메모 | `venue_specs` 텍스트 (`이 행사장은 ~`) |
| **B. 발주 완료** | 사용자가 편집기에서 **엑셀 내보내기** 클릭 | 그 시점의 design_items 17컬럼 전체 | 가중치 100% 정답 풀에 편입 |
| **C. 예외 입력 (시설 가이드 위반 후 ′그래도 진행′)** | 시설 가이드 알랏에서 사용자가 표준 외 입력을 의식적으로 통과시킬 때 | 위반 항목 + ′예외 케이스′ 태그 | ′우리 매뉴얼 정보가 틀렸을 가능성′ 학습 풀 (관리자 검토 후 매뉴얼 갱신 권고) |

→ A·B·C 모두 **텍스트 파일 형태**로 누적. AI가 추천 시점에 ′이 행사장이다 + 이 행사 유형이다 + 도면 패치 정보′ 세 가지를 함께 받아 분석.

## 3. 시설 가이드 동작 (사용자 안내 형식)

### 3-1. 두 가지 경로
- **자동 알랏**: 사용자가 표준 외 규격·재질 등을 입력하면 첫 1회만 팝업으로 안내 (강제 차단 X)
- **수동 조회**: 에디터 우상단 `[행사장 가이드 보기]` 버튼 클릭 시 슬라이드 패널로 조회

### 3-2. 알림 강도 3단 토글 (프로젝트별)
| 모드 | 알랏 팝업 | 그리드 ⚠️ 아이콘 | 다운로드 직전 일괄 요약 |
|---|---|---|---|
| `verbose` (기본) | ✅ 첫 위반 1회 | ✅ 상시 | ✅ |
| `silent_icon` | ❌ | ✅ 상시 | ✅ |
| `off` | ❌ | ❌ | ✅ (안전망) |

### 3-3. ′그래도 진행′ 데이터 처리
- 비고에 자동으로 `[시설 가이드 외] 사용자 확인 진행` 표기
- ′예외 케이스′ 태그로 별도 저장 → 위 §2-C 학습 풀로 편입
- 같은 행사장에서 예외가 반복되면 관리자에게 ′매뉴얼 갱신 권고′ 자동 알림

## 4. 신규 행사장 → 추천까지의 전체 흐름

```
[사용자] 행사장 드롭다운에 없음 → ′신규 행사장 등록 요청′ 클릭
   ↓
[모달] 행사장명 + 권역 + 유형 + (선택) 도면 첨부
   ↓
[venue_requests INSERT] status='pending'
   ↓
[관리자 (/admin/learning)] 요청 검토 → 승인
   ↓
[venues INSERT + learning_jobs INSERT (queued)]
   ↓
[백엔드 AI 분석] 도면·정보 분석 → 텍스트 파일 형태로 venue_specs에 저장
   ↓
[새 프로젝트 생성 시] 드롭다운에 해당 행사장 즉시 노출 + AI가 venue_specs 컨텍스트로 추천 산출
```

## 5. 회의록 추가 결정

- 위자드 3단계(제작물 선택) **홀딩** — 목적성 미확정, 팀장 논의 후 결정. 2단계(팀원 초대) 완료 시 바로 편집기 진입
- 1단계 발주서 엑셀 입력 **드랍** — ″이미 학습한 걸로 추천하는 시스템인데 왜 엑셀?″
- 관리자 페이지 KPI에서 **실적 매칭·PM 사업부·발주처·행사 폴더 4개 제거** — 사용 안 하기로
- 행사 분류 ′추천 핵심′ 같은 사용자에게 불필요한 라벨 **제거**
- 팀원 초대: PRS 연동 가능성 있으나 미구현. 일단 **이름 + 회사 이메일 + 담당 파트 수기 입력** (모니터링 용도)

## 6. 편집기 가로/세로 분할 — 검토 항목

- 현재: 가로 분할 (상단 그리드 50% + 하단 캔버스 50%)
- 회의 의견: ″디자인을 별로 안 보니까 세로로 잡으면 빨리 끝난다″ — 좌측 그리드 + 우측 캔버스 형태 검토
- 결정 보류 → 사용자 테스트 후 결정
