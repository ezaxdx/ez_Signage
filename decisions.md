```

**선택**: <무엇을 골랐나>
**대안**: <고려한 다른 옵션>
**이유**: <왜 그걸 골랐나>
**되돌릴 수 있는가**: <쉬운가, 어려운가>
```

## 2026-04-24 — 캔버스 렌더러: Fabric.js v5

**컨텍스트**: 제작물 미리보기 + 슬롯 드래그 위치 조정 구현 필요
**선택**: Fabric.js v5
**대안**: Konva.js, plain Canvas API
**이유**: IText 더블클릭 직접 타이핑, object:modified 이벤트, 좌표 % 변환이 충분히 검증됨
**되돌릴 수 있는가**: 어려움. Fabric.js 의존 코드가 캔버스 전체에 퍼져 있음.

## 2026-04-24 — 좌표계: 0~100% 정규화

**컨텍스트**: Gemini bbox(0~1000)와 캔버스(px) 사이 변환 필요
**선택**: 앱 내부는 0~100% 정규화, PPT 출력 시 inch로 변환
**대안**: 0~1000 그대로 유지
**이유**: CSS % 값과 직접 연동 가능, 규격(mm) 변경에 무관
**되돌릴 수 있는가**: 가능하나 전체 좌표 코드 수정 필요.

## 2026-04-24 — DB: Supabase PostgreSQL + RLS

**컨텍스트**: 멀티유저 실시간 협업, 프로젝트별 접근 제어 필요
**선택**: Supabase (PostgreSQL + Realtime + Storage + Auth)
**대안**: Firebase Firestore (banner_gen_os 프로토타입에서 사용)
**이유**: RLS로 row-level 보안, item_contents Realtime 구독, 추후 데이터허브 SSO 연동 고려
**되돌릴 수 있는가**: 어려움. 마이그레이션 비용 큼.
