# /meeting-prep <YYMMDD>

회의록 작성 4 단계 표준 워크플로우 (5/18 김연아 대리 카톡 지적 후 영구 영역).

## 실행 절차 (4 단계)

### 1단계 — 입력 자료 자동 식별

- 회의 날짜 영역 입력받음 (예: 5/18 → 260518)
- `업무/회의록/회의록 녹음 TXT/` 영역 자동 검색 = `YYMMDD_*.txt` 파일 매칭
- 매칭 X = 사용자에게 STT 파일·메모 영역 요청 후 작성 시작 X

### 2단계 — 본문 작성 (입력 자료 영역만)

- 매 항목 = STT 또는 메모에 있는 영역만 사용
- 내부 메모리·CLAUDE.md 박제 룰 본문 추가 X (양식·댑스·맞춤법만 영향 OK)
- 영어 약어 (SOT·BLUF·MECE·KPI·ROI·DB·UI·API 등) 자체 노출 X·한국어 실무 표현 대체
- 실명 확인 = `새 폴더\사내연락망\` 최신 파일 자동 참조

### 3단계 — 작성 후 자체 검증

- 매 항목 = STT 영역 grep·매칭 X 영역 = 사용자에게 명시·승인 후 유지·기본 = 제거
- 영어 약어 검색·AI 어투 점검 (표·기호·균일 패턴 절제)
- "입력 자료 외 추가된 영역 N건·각 출처" 사용자 명시

### 4단계 — 사용자 컴펌

- 사용자 영역별 승인·반려 후 단톡 공유
- 분류 = 회의록 = `[단순 보고]` 또는 `[검토 요청]` 첫 줄 명시

## 양식

- 주간 회의록 = 메모리 `reference_주간회의록_양식_SOT_260518` SOT
- 일반 회의록 = `새 폴더\회의\회의 양식\예시 (1).docx`
- 아웃풋 위치 = `업무/회의록/회의록 아웃풋/`
- 파일명 = `회의록_<주제>_<YYMMDD>.docx`

## 출력 후

- md_to_docx.ps1 자동 호출 = docx 변환
- 검증 = 입력 자료 grep matching·실명 표기 정확

## 금지

- STT 자료 없으면 회의록 작성 시작 X
- 메모리·CLAUDE.md 룰 본문 추가 X
- 영어 약어 본문 자체 노출 X
- "97%·완벽" 표현 X
