# /learning-status

학습 풀 9개 영역 현황 빠른 출력. 사용자가 "지금 학습 어떻게 됐어?" 물을 때 즉시 답.

## 실행 절차

1. `npm run check:morning` 학습 풀 섹션 추출
2. 9개 영역 시드 카운트 grep
3. 누락 영역 4건 식별 (천정배너·시설 가이드 골격·시드 통합·hall 매핑)
4. 다음 보강 시점 명시

## 출력 형식

### 1. 시드 카운트 (9 영역)
| 영역 | 카운트 |
|---|---|
| 카테고리 | 12종 |
| 12파트 매핑 | 12 파트 |
| 행사 학습 인덱스 | N 행사·M 파일 |
| 행사장 L1 | N 행사장 |
| 홀 L2 | N 홀 |
| 시설 가이드 | N venue (실측 5·골격 N) |
| 천정배너 패턴 | N 행사장 |
| 6대 표준 카테고리 | 외벽·천정·게이트·가로등·X배너·부속시설 |
| 동의어 | N 매핑 |

### 2. 강 영역 / 약 영역
- 강 = 코엑스·그랜드하얏트·송도·킨텍스
- 약 = 시설 가이드 21건 골격만·천정배너 코엑스·송도 누락

### 3. 다음 보강
- 5/22 라이브 후 1~2주 = 실 사용자 데이터 누적
- 5/25 D+3 = 메모리 archive·SectionBoundary wrap
- 5/26 D+4 = 시드 SOT 통합·harness Iteration 4
